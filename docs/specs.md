# Spec-Driven Development (SDD) Protocol

For any new feature or query requirement added to this project:

## Process Flow
1. **Requirements (`requirements.md`)**: Define features using EARS-BI syntax (Ubiquitous, Event-Driven, Unwanted Behavior, State-Driven).
2. **Design (`design.md`)**: Outline SQL logic, CTE structures, data validation, and safety handling.
3. **Tasks (`tasks.md`)**: Detail implementation steps.
4. **Approval**: Obtain human approval before starting implementation.
